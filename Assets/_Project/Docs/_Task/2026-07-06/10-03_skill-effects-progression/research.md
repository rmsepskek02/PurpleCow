# Research - Skill Effects and Character Progression

이번 작업은 삼택지에서 고른 액티브/패시브 스킬이 실제 볼 외형과 전투 수치에 반영되도록 만들고, 캐릭터 경험치 곡선과 최대 레벨을 6개 스킬 × 3레벨 구조에 맞게 다시 설계하기 위한 조사이다. 공식 과제 PDF, 현재 데이터 에셋, 런타임 코드, 기존 작업 문서를 서로 대조했다.

## 현재 상태

### 캐릭터 경험치와 최대 레벨

- `CharacterManager`의 필요 경험치는 `{ 10, 20, 30 }` 세 값뿐이다.
- 기본 몬스터 보상은 10 XP, 2칸 몬스터 보상은 18 XP다. 따라서 첫 기본 몬스터 한 마리만 처리해도 즉시 Lv.2가 되어 게임 시작 직후 삼택지가 열린다.
- 필요 경험치 배열을 모두 소비하면 `RequiredXp`가 0이 되며 더 이상 경험치를 받지 않는다. 시작 레벨이 1이므로 현재 최대 캐릭터 레벨은 4다.
- 20웨이브의 스폰 수는 총 295마리이며, 현재 종류별 등장 가중치와 보상을 적용한 전체 기대 경험치는 약 3,940 XP다.
- 장착 한도는 액티브 4개 + 패시브 2개로 총 6개이고 각 스킬은 Lv.3까지 존재한다. 모든 슬롯을 채우고 모두 Lv.3으로 만들려면 총 18회의 선택이 필요하다.
- 캐릭터가 Lv.1에서 시작하고 레벨업마다 한 번 선택한다면, 18회 선택을 모두 제공하는 최대 캐릭터 레벨은 **Lv.19**가 된다.

### 스킬 레벨 데이터

- `SkillData`에는 레벨별 볼 데미지와 효과 수치가 모두 들어 있다.
- 현재 `_currentLevel`도 `SkillData` ScriptableObject 자체에 저장하고 런타임에 변경한다. 이는 `DevRules.md`의 “SO 원본 데이터는 읽기 전용, 런타임 수치는 별도 런타임 클래스에서 관리” 규칙과 어긋난다.
- `_currentLevel == 0`이 “미보유”와 “보유 Lv.1” 양쪽 의미로 사용된다. 실제 보유 여부는 `SkillManager` 목록으로 따로 판정하지만, `SkillCardUI`는 데이터의 레벨만 보고 `New!`와 표시 레벨을 결정한다.
- 그 결과 이미 보유한 Lv.1 스킬의 업그레이드 카드도 `New! / Lv.1`로 보일 수 있으며, 실제 선택 후에는 Lv.2가 된다.
- 액티브/패시브 스킬이 레벨업한 경우 변경 이벤트를 다시 발행하지 않는 코드 경로도 있다.

### 액티브 볼 외형과 기본 데미지

- 전용 리소스는 이미 모두 존재한다.
  - `Ball_Nomal_Ball.png`
  - `Ball_Fire_ball.png`
  - `Ball_Ice_Ball.png`
  - `Ball_Ghost_Ball.png`
  - `Ball_Laser_Ball.png`
  - `Ball_Cluster_Ball.png`
- 현재 모든 볼은 하나의 `Ball.prefab`을 풀링해서 사용하며 항상 노멀 볼 SpriteRenderer 외형을 유지한다.
- `BallLauncher`의 로스터는 신규 액티브 선택 시 특수 볼을 한 개 추가하고, 재선택 시 볼 개수는 늘리지 않고 해당 스킬 레벨만 올린다. 이 로스터 구조 자체는 기존 확정 규칙과 일치한다.
- 가장 큰 누락은 `SkillLevelData.BallDamage`다. 카드에는 11~50의 액티브 볼 공격력이 표시되지만, 실제 `Ball.CalculateDamage()`는 모든 메인 볼에 `BallData.Damage` 8만 사용한다.
- 따라서 현재 액티브 볼은 전용 외형도 없고 레벨별 기본 공격력도 적용되지 않는다.

### 액티브 스킬 효과

공식 PDF의 수치 데이터는 에셋에 정확히 입력되어 있으나 일부 런타임 계산이 잘못되었거나 완성되지 않았다.

| 스킬 | PDF 의도 | 현재 구현 상태 |
|---|---|---|
| 파이어 볼 | 타격마다 화상 1스택 추가, 최대 3/4/5스택, 스택마다 초당 8/10/12 피해, 4/4.5/5초 | 첫 타격부터 `maxStacks`를 현재 스택처럼 사용해 최대 중첩 피해를 즉시 가함. 실제 스택 누적/갱신이 없음 |
| 아이스 볼 | 30/35/40% 확률로 5/6/7초 냉동, 이동속도 10/15/20% 감소, 추가 피해 10/15/20% | 확률·시간·감속·추가 피해 코드는 존재. 다만 기본 볼 데미지가 연결되지 않아 전체 결과가 틀림 |
| 레이저 볼 | 같은 행의 모든 적에게 7/11/15 추가 피해 | 대상 외 같은 행 몬스터에 추가 피해를 주는 코드가 존재 |
| 고스트 볼 | 몬스터를 관통 | 트리거 전환으로 관통하는 코드가 존재 |
| 클러스터 볼 | 타격 시 40/50/60% 확률로 10/15/20 피해 특수 볼 생성 | 확률과 서브 볼 1개 생성 코드는 존재. 서브 볼 외형은 현재 노멀 볼이며 생성된 볼에는 클러스터 정체성이 없음 |

추가로 파이어 DOT 코루틴은 풀에 반환된 몬스터에서 명시적으로 정리되지 않으므로, 같은 오브젝트가 재사용될 때 이전 화상 상태가 새 몬스터 생명주기에 섞일 위험이 있다.

### 패시브 스킬 효과

| 스킬 | PDF 의도 | 현재 구현 상태 |
|---|---|---|
| 따뜻한 양철 심장 | 모든 **노멀 볼** 추가 피해량 20/30/40% 증가 | 전역 데미지 배율로 구현되어 특수 볼에도 적용됨 |
| 마법 거울 | 볼이 벽을 칠 때마다 다음 타격 피해량 20/40/60% 증가 | 0.2/0.4/0.6을 “고정 피해”로 더함. 여러 볼의 벽 충돌 보너스가 전역 누적되고 아무 볼의 다음 타격이 모두 소비함 |
| 자수정 단검 | 적 전면 타격 시 해당 타격 치명타 확률 10/20/30% 증가 | 데미지/치명타 판정이 끝난 뒤 보너스를 대상 몬스터에 저장하여 현재 타격이 아니라 다음 타격에 적용됨 |
| 에메랄드 단검 | 적 후면 타격 시 해당 타격 치명타 확률 20/30/40% 증가 | 자수정 단검과 같은 적용 순서 문제 |
| 마지막 성냥 | 적 사망 시 폭발해 근처 적에게 10/20/30 피해 | 주변 피해 코드는 존재하며 레벨 수치도 연결됨 |

## 공식 PDF와 기존 규칙 대조

- 공식 PDF는 액티브 최대 4개, 패시브 최대 2개, 각 스킬 최대 Lv.3을 요구한다.
- 같은 스킬은 신규 카드로 중복 등장하지 않고, 보유한 스킬은 레벨업 후보로 등장해야 한다.
- 노멀 볼 기본 충돌 데미지는 8, 기본 치명타 확률은 0%, 기본 치명타 배율은 50% 추가다. 현재 `BallData.asset`은 8 / 0 / 1.5로 일치한다.
- PDF의 구현 제외 항목인 선택지 다시 뽑기, 융합 시스템, 보스 등은 이번 작업에 포함하지 않는다.
- PDF는 “각 웨이브에서 조건 충족 시 삼택지”를 설명하지만, 현재 프로젝트와 사용자 요청은 캐릭터 XP 레벨업을 삼택지 트리거로 사용한다. 이번 작업은 사용자 요청에 따라 기존 XP 기반 흐름을 유지한다.

## 구현 전에 확정이 필요한 사항

1. **최대 캐릭터 레벨**
   - 권장안: 시작 Lv.1 + 선택 18회 = **최대 Lv.19**
   - 이 경우 6개 스킬을 모두 Lv.3까지 올릴 수 있다.

2. **필요 경험치 곡선**
   - 권장안: Lv.2 필요 XP 50에서 시작해 이후 레벨마다 18씩 증가
   - 실제 배열: `50, 68, 86, ... , 338, 356` (18구간, 총 3,654 XP)
   - 첫 삼택지는 기본 몬스터 기준 약 5마리 처리 후 열리고, 20웨이브 전체 기대 XP 약 3,940 안에서 Lv.19 도달이 가능하다.

3. **마법 거울 적용 단위**
   - 권장안: 벽에 충돌한 **그 볼의 다음 몬스터 타격 1회**에 퍼센트 배율을 적용하고 소비한다.
   - 전역 공유 방식은 다중 볼 게임에서 어떤 볼이 보너스를 가져가는지 예측하기 어렵다.

4. **클러스터 서브 볼 외형**
   - 권장안: 생성한 서브 볼에도 `Ball_Cluster_Ball.png`를 사용한다.
   - 서브 볼은 지정된 10/15/20 고정 피해만 주고, 추가 클러스터 생성 효과는 다시 부여하지 않아 연쇄 생성을 막는다.

## 결론

현재 시스템은 삼택지 UI와 보유 슬롯, 특수 볼 로스터의 뼈대는 갖춰져 있다. 이번 구현의 핵심은 `SkillData`를 읽기 전용 정의로 되돌리고 런타임 레벨을 분리한 뒤, 특수 볼별 Sprite와 `BallDamage`를 각 로스터 볼에 연결하는 것이다.

그와 함께 파이어 중첩, 노멀 볼 전용 피해 증가, 마법 거울의 퍼센트/소유 볼 처리, 단검 치명타 판정 순서를 공식 PDF 설명대로 바로잡아야 한다. 캐릭터 최대 레벨과 XP 곡선, 마법 거울 및 클러스터 볼의 적용 단위를 사용자에게 확정받은 후 상세 구현 계획을 작성한다.
