# TaskRules.md

이 문서는 프로젝트에서 task 문서를 작성하고 관리하는 규칙을 정의합니다.
모든 작업은 이 규칙을 따라 문서화되며, AI와 사용자 간의 명확한 소통을 목적으로 합니다.

---

## 폴더 구조

```
Assets/_Project/Docs/_Task/
└── YYYY-MM-DD/
    └── HH-MM_작업요약/
        ├── research.md
        └── plan.md
```

**예시:**
```
Assets/_Project/Docs/_Task/
└── 2026-06-30/
    └── 14-30_볼발사로직구현/
        ├── research.md
        └── plan.md
```

---

## 작업 흐름

```
논의 완료
    ↓
task 문서 작성 여부 확인 (문서 작성 작업 제외)
    ↓
research.md 작성 → 사용자 확인
    ↓
plan.md 작성 → 사용자 확인
    ↓
사용자가 명시적으로 승인 → 구현 시작
```

- 사용자가 "진행해"라고 해도 task 문서 작성 여부는 항상 먼저 확인한다
- plan.md 작성 후 사용자의 명시적인 승인 없이는 절대 구현하지 않는다
- 문서 작성 작업 자체에는 task 문서를 작성하지 않는다

---

## research.md

현재 프로젝트 상태를 파악하고 원인을 분석하는 문서.
AI가 파악한 내용을 사용자와 상세하게 공유하기 위한 목적으로 작성된다.

**구조:**
```
# Research — [작업명]

[문서 서두: 이 문서가 무엇을 다루는지 자연어로 2~4줄 요약]

## 현재 상태
## 관련 파일 및 의존성
## 문제점 / 구현 대상 파악
## 결론
```

---

## plan.md

research.md를 기반으로 구현 계획을 다루는 문서.
AI가 구현할 내용을 사용자와 상세하게 공유하기 위한 목적으로 작성된다.

**구조:**
```
# Plan — [작업명]

[문서 서두: 이 문서가 무엇을 계획하는지 자연어로 2~4줄 요약]

## 구현 목표
## 단계별 작업 계획
## 예상 변경/생성 파일 목록
## 주의사항
```
