# GameplayMechanics.md

이 문서는 UI 규칙이 아니라 "게임 내 구체적인 알고리즘/메커닉"을 기록하는 문서입니다.
UI 배치나 연출이 아닌, 게임플레이 로직 자체의 동작 방식(조준, 발사, 충돌, 귀환 등)을 다룹니다.
향후 이런 종류의 알고리즘/메커닉이 추가로 확정되면 이 문서에 섹션을 계속 추가합니다.
원본 게임(통통 디펜스: 핀볼 마스터) 실제 플레이 경험을 보유한 사용자가 확인해준 내용을 기준으로 작성되었습니다.

---

## 1. 볼 발사 및 궤도 시스템

원본 게임 실제 플레이어(사용자)가 확인해준 볼 발사/궤도 메커닉입니다.

- 플레이어가 화면을 **터치하는 순간** 조준 방향이 즉시 정해진다. 드래그 후 손을 떼는 시점이 아니라, 터치를 시작하는 시점에 조준이 시작된다.
- 조준 중에는 점선 형태의 예상 경로가 표시되며, 정확히 2단계 구간까지만 그려진다. 전체 반사 경로 전부를 보여주는 것이 아니다.
  1. 조준 방향으로 뻗어나가 **첫 번째 충돌 지점**까지 점선이 그려진다.
  2. 그 지점에서 **반사(튕겨나가는) 방향**으로 이어져 **두 번째 충돌 지점**까지 점선이 계속 그려진다.
  3. 두 번째 충돌 지점에는 **빨간 점(레드닷)**과 그 점을 감싸는 **둥근 원형 궤적선(고리 형태)**이 표시된다.
  4. 그 이후(세 번째 충돌 이후)의 경로는 표시되지 않는다.
- 드래그하면 궤적이 드래그 위치를 목표로 실시간으로 따라간다. 터치를 유지하는 동안 계속 갱신된다.
- 한 번 발사되어 날아간 볼은 플레이어가 더 이상 직접 제어할 수 없다. 벽/몬스터와의 충돌에 따라 물리적으로 방향이 바뀌며 이동한다.
- 볼은 최종적으로 화면 하단에서 튕겨 캐릭터 위치로 돌아온다.
- 캐릭터에게 돌아온 볼은 **그 시점의 최신 조준 방향**으로 자동으로 다시 발사된다. 재발사 시점에 정해진 방향을 사용하며, 최초 발사 당시의 방향이 아니다.
- 이 발사 → 충돌 이동 → 귀환 → 재발사 사이클이 계속 반복되며, 플레이어는 이 사이클 속에서 "다음 재발사 때 사용할 조준 방향"을 실시간으로 계속 조준할 수 있다.
- 볼의 이동 속도는 조준/조작과 무관하게 항상 일정하다.
