# ReferenceUI 이해

이 문서는 원본 게임(통통 디펜스: 핀볼 마스터, CookApps, `com.cookapps.bouncedefense`)의 실제 플레이 스크린샷 4장을 사용자(실제 플레이 경험 보유자)와 함께 분석하여 확정한 UI/게임플레이 이해를 정리합니다.
PurpleCow_클라이언트_채용과제.pdf가 공식 요구사항 스펙이자 최우선 기준이며, 이 문서는 그 스펙을 보완하는 레퍼런스 자료입니다.
스크린샷에 등장하더라도 PDF가 "구현 제외 항목"으로 명시한 기능은 이번 프로토타입 구현 범위에서 완전히 제외됩니다.
향후 UI/게임플레이 작업 범위와 구현 방향을 정하는 기준 자료로 사용됩니다.

---

## 1. 배경

- **공식 스펙**: PurpleCow_클라이언트_채용과제.pdf가 최우선 기준이다.
- **레퍼런스**: 원본 게임(통통 디펜스: 핀볼 마스터, CookApps, `com.cookapps.bouncedefense`)의 실제 플레이 스크린샷 4장을 분석했다. 스크린샷 이미지 파일 자체는 아직 프로젝트에 추가되지 않았으며, 이 문서에는 텍스트 설명만 기록한다. 이미지가 추가되면 [11. 레퍼런스 이미지](#11-레퍼런스-이미지) 섹션에 경로를 채워 넣는다.
- **PDF상 구현 제외 항목**: 튜토리얼, 배속 기능, 1스테이지 보스, 자동 조준 기능, 선택지 다시뽑기(리롤), 융합 시스템. 스크린샷에 등장하더라도 이번 프로토타입에서는 전혀 구현하지 않는다.

---

## 2. 스테이지 진행률 바

- 화면 상단의 진행률 바는 현재 스테이지의 진행률을 나타내는 것이 맞다.
- 바 끝에 있는 아이콘은 보스 아이콘이며, 원본 게임에서는 게이지가 다 차면 보스가 등장하는 구조다.
- 이번 프로토타입은 보스를 구현하지 않으므로, 보스 아이콘과 보스 자체는 구현하지 않고 **진행률 표시 기능만** 구현한다.

---

## 3. 처치 게이지 / 레벨 배지 (우측 상단 방패 모양 배지 + 주황 게이지)

- 이 UI는 경험치와 레벨 개념이다. 몬스터를 처치하면 게이지가 채워지고, 게이지가 가득 찰 때마다 삼택지(3택지) 스킬 선택 UI가 노출된다.
- 현재 코드(`WaveManager.cs`)의 "누적 처치 수가 임계값의 배수에 도달하면 이벤트 발행" 로직 자체는 이 개념과 부합한다.
- 다만 현재 HUD는 "처치: N" 형태의 누적 텍스트로만 표시되고 있어, 경험치 게이지/레벨 배지 형태의 시각화가 없다. **이 부분은 UI 재설계가 필요한 갭으로 남아있다.**

---

## 4. 플레이어 HP 바

- 하단 캐릭터 HP 바는 실제 체력(HP)이 맞으며, 몬스터가 화면 하단에 도달하면 감소한다.
- 이는 현재 코드(`CharacterManager.cs`가 `WaveManager.OnMonsterReachedBottom` 이벤트를 구독해 `TakeDamage` 호출)의 동작과 일치한다.
- 다만 현재는 Slider(막대)만 있고, 스크린샷처럼 정확한 숫자(예: 230/258/300)를 보여주는 텍스트 표시가 없다. **이 부분도 갭으로 남아있다.**

---

## 5. Auto 버튼

- 원본 게임 화면에 있는 "Auto" 버튼은 PDF의 자동 조준 기능 제외 규정에 해당하며, 이번 프로토타입 구현 범위에서 생략한다.

---

## 6. 탄도 예측선 (조준 궤적)

- 발사 전에 예상 경로가 점선으로 표시되고, 그 경로대로 볼이 발사된다.
- 또한 볼이 비행하는 도중에 플레이어가 다시 화면을 터치하면, 터치한 방향으로 볼의 경로가 재조정(redirect)되는 방식이다.
- 즉 단순 조준선 프리뷰뿐 아니라 **"비행 중 재조준" 기능까지 포함하는 메커닉**이다.
- 현재 코드(`InputHandler.cs`, `BallLauncher.cs`)에 궤적 프리뷰나 비행 중 재조준 로직이 있는지는 별도 확인이 필요하다. **(미확정, 추후 코드 검토 필요)**

---

## 7. 데미지 표시 방식

데미지는 크게 두 가지 방식으로 구분된다.

1. 볼이 몬스터와 충돌했을 때 즉시 발생하는 일반 데미지
2. 특수볼에 의한 지속 데미지(DOT)

PDF 상 액티브 스킬 5종(파이어볼/아이스볼/레이저볼/고스트볼/클러스터볼) 중 지속 데미지(DOT) 방식은 **파이어볼**이 유일하다.

| 항목 | 수치 |
|------|------|
| 방식 | 화상 중첩 |
| 최대 중첩 (레벨별) | 3 / 4 / 5중첩 |
| 지속 시간 (레벨별) | 4 / 4.5 / 5초 |
| 초당 피해 (레벨별) | 8 / 10 / 12 |

나머지 스킬(아이스볼/레이저볼/고스트볼/클러스터볼)은 즉시 데미지 또는 확률 기반 즉시 처리 방식이다.

현재 코드의 `FireBallSkill.cs`에는 이미 DOT 로직(`ApplyDot` 호출)이 구현되어 있어, PDF 기준으로는 **추가 작업이 필요 없는 상태**로 확인됐다.

---

## 8. 필드 위 부유 아이템 (다이아몬드 젬처럼 보였던 것)

- 이는 별도의 전용 에셋이 아니라, 볼(특수볼) 자체를 가리키는 것으로 추정된다.
- 프로젝트에는 이미 볼 종류별 스프라이트 6종(`Ball_Nomal_Ball`, `Ball_Cluster_Ball`, `Ball_Ice_Ball`, `Ball_Fire_ball`, `Ball_Ghost_Ball`, `Ball_Laser_Ball`)이 `Assets/_Project/Sprites/Ball/`와 `Assets/_Project/Sprites/BallSkillIcon/`에 준비되어 있어, **신규 에셋 제작은 필요하지 않을 것**으로 판단된다.

---

## 9. "Best!" 추천 아이콘

- 레벨업 스킬 선택 화면에서 특정 카드에 붙는 추천 표시 아이콘은 원본 게임에 실제로 존재하는 기능이다.
- 다만 프로젝트 리소스(Sprites, Resources, 스크립트 키워드 검색)를 전수 확인한 결과 관련 에셋이 전혀 없다.
- **리소스가 준비되기 전까지는 이번 구현 범위에서 제외한다.**

---

## 10. 정리 — 현재 확인된 미구현 갭 (재확인/후속 작업 필요)

| 갭 | 현재 상태 |
|----|-----------|
| 경험치 게이지/레벨 배지 형태의 HUD | 누적 처치 수 텍스트만 존재 |
| 플레이어 HP 숫자 텍스트 표시 | Slider만 존재 |
| 스테이지 진행률(%) 표시 UI (보스 아이콘 제외) | 미구현 |
| 스킬 카드에 볼 데미지 수치 표시 | `SkillData.CurrentLevelData.BallDamage` 연동 필요 |
| Active 4 / Passive 2 슬롯의 시각적 UI | 내부 로직은 있으나 화면 표시 없음 |
| 발사 전 탄도 예측선 + 비행 중 재조준 기능 | 여부 확인 및 필요 시 구현 |

**참고 (구현하지 않는 항목)**
- Auto 버튼, 리롤, 보스, 배속, 튜토리얼, 융합 시스템은 PDF 제외 항목이므로 구현하지 않는다.
- "Best" 추천 아이콘은 리소스 부재로 구현하지 않는다 (리소스 준비 시 재검토).
