# PurpleCow - 통통 디펜스: 핀볼 마스터 1스테이지 카피

CookApps 클라이언트 프로그래머 채용 과제로 제작한 Unity Android 프로젝트입니다.  
원본 게임의 1스테이지 핵심 플레이를 분석해 핀볼 전투, 웨이브, 로그라이크 스킬 선택, UI와 결과 처리를 구현했습니다.

## 개발 환경

- Unity `6000.3.10f1`
- Target Platform: Android
- Render Pipeline: Universal 2D URP
- Input: Unity New Input System
- UI: TextMeshPro
- Tween: DOTween
- 기본 씬: `Assets/Scenes/SampleScene.unity`

## 실행 방법

1. Unity Hub에서 프로젝트를 Unity `6000.3.10f1`로 엽니다.
2. `Assets/Scenes/SampleScene.unity`를 엽니다.
3. Play Mode를 실행합니다.

씬과 데이터 연결은 저장소에 반영되어 있으므로 Setup 메뉴를 다시 실행할 필요가 없습니다.

Android 빌드는 Unity의 Build Profiles에서 Android를 선택하고 `SampleScene`을 포함해 진행합니다.

## 조작법

- 화면 터치/드래그: 발사 방향 조준
- 볼: 발사 후 자동으로 반사·귀환하며 최신 조준 방향으로 순차 재발사
- 우측 상단 `Ⅱ`: 일시정지
- 우측 하단 첫 번째 버튼: 스피드업
- 우측 하단 두 번째 버튼: 분신

## 구현 내용

### 핀볼 전투

- 노멀 볼 5개로 시작
- 몬스터·벽·바닥 충돌과 반사
- 2단계 반사 궤적 프리뷰
- 볼 귀환 후 FIFO 순차 재발사
- 볼 사이의 물리 충돌 방지
- 오브젝트 풀을 이용한 볼 재사용

### 웨이브와 몬스터

- 총 20웨이브
- Fluffy, Spider, StoneBug, ForestDeer 4종
- 1×1, 2×1, 1×2 블록 크기
- 웨이브 진행에 따른 스폰 수와 2칸 몬스터 비중 증가
- 빈 그리드 셀을 검사하는 컨베이어식 스폰
- 몬스터 HP, 하강, 바닥 도달 데미지

### 캐릭터 성장과 3택지

- 캐릭터 Lv.1~Lv.19
- Lv.2 필요 경험치 50, 이후 레벨마다 요구량 18 증가
- 레벨업 시 액티브/패시브 통합 3택지
- 액티브 최대 4종, 패시브 최대 2종
- 각 스킬 최대 Lv.3
- 신규 액티브 획득 시 전용 특수 볼 추가
- 동일 스킬 재선택 시 볼 개수는 유지하고 레벨만 상승

### 액티브 볼 스킬

- 파이어 볼: 중첩 화상 지속 피해
- 아이스 볼: 확률 냉동, 이동속도 감소, 추가 피해
- 레이저 볼: 같은 행 몬스터 추가 피해
- 고스트 볼: 몬스터 관통
- 클러스터 볼: 확률적으로 특수 서브 볼 생성

특수 볼은 전용 Sprite와 레벨별 기본 공격력을 사용합니다.

### 패시브 스킬

- 따뜻한 양철 심장: 노멀 볼 피해 증가
- 마법 거울: 벽 충돌 후 해당 볼의 다음 타격 피해 증가
- 자수정 단검: 전면 타격 치명타 확률 증가
- 에메랄드 단검: 후면 타격 치명타 확률 증가
- 마지막 성냥: 몬스터 사망 시 주변 폭발 피해

### 플레이어 액티브 스킬

과제 필수 범위와 별도로 추가한 가산점 콘텐츠입니다.

- 스피드업
  - 쿨타임 30초
  - 6초 동안 모든 볼 속도 1.5배
  - 지속 중 새로 발사되는 볼에도 적용
- 분신
  - 쿨타임 30초
  - 현재 원본 볼 로스터를 복제해 순차 발사
  - 복사 볼은 두 번째 회수 시 제거

### UI

- 원본 플레이 캡처 기반 HUD
- 스테이지 진행률, XP, 캐릭터 레벨
- 캐릭터 현재/최대 HP
- 액티브/패시브 보유 슬롯
- 레벨업 3택지 카드
- 일시정지 패널
- 성공/실패 결과 팝업과 재시작
- Safe Area와 다양한 세로형 화면 비율 대응
- 모든 게임 텍스트에 `Maplestory Bold SDF` 적용

## 주요 기술 설계

### 타입별 볼 로스터

`BallLauncher`가 노멀 볼과 획득한 특수 볼을 개별 로스터로 관리합니다. 각 볼은 자기 타입과 스킬 상태를 유지하며 귀환 순서대로 재발사됩니다.

### 읽기 전용 데이터와 런타임 상태 분리

스킬·볼·몬스터·웨이브 설정은 ScriptableObject로 관리합니다. 스킬의 현재 레벨처럼 플레이 중 변하는 값은 `SkillRuntimeState`로 분리해 원본 에셋을 변경하지 않습니다.

### 오브젝트 풀

볼과 몬스터를 `ObjectPool<T>`로 재사용해 반복 생성과 파괴를 줄였습니다. 풀 반환 시 화상, 냉동, Sprite, 스킬 등 이전 생명주기의 상태를 초기화합니다.

### 해상도 대응

배경 격자의 고유 가로세로 비율을 보정하고, `BackgroundFitter`, `WallFitter`, `SafeAreaFitter`가 배경·벽·발사 지점·UI를 기기 화면에 맞춰 배치합니다.

### 입력 분리

터치 시작 지점이 UI인지 판정해 액티브 스킬 버튼이나 일시정지 버튼 입력이 볼 조준으로 동시에 처리되지 않도록 분리했습니다.

## 구현 제외 항목

공식 과제 PDF의 제외 범위에 따라 다음 기능은 구현하지 않았습니다.

- 튜토리얼
- 배속 기능
- 1스테이지 보스
- 자동 조준
- 선택지 다시 뽑기
- 융합 시스템
