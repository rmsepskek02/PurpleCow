# CLAUDE.md

이 문서는 모든 에이전트가 공통으로 따라야 할 행동 지침입니다.
개발 에이전트 전용 규칙은 [DevRules.md](Assets/_Project/Docs/DevRules.md)를 참고하세요.

---

## 0. 작업 전 사용자 확인 (User Confirmation First)

- 모든 작업(스크립트 생성, 설정 변경, 삭제 등)을 진행하기 전에 **자연어로 상세히 설명**한다
- 무엇을 왜 어떻게 할 것인지 먼저 설명하고, 사용자의 판단을 기다린다
- 설명 없이 바로 파일을 생성하거나 수정하지 않는다
- 문서 작성 작업은 사용자 확인 후 바로 진행한다
- task 문서 관련 규칙은 [TaskRules.md](Assets/_Project/Docs/TaskRules.md) 참고

## 1. task 문서 작업 흐름

- 스크립트 생성, 설정 변경, 삭제 등 수정을 요하는 작업 전 task 문서 작성 여부를 항상 확인한다
- 문서 작성 작업 자체에는 task 문서를 작성하지 않는다
- plan.md 작성 후 사용자의 명시적인 승인 없이는 절대 구현하지 않는다
- 사용자가 "진행해"라고 해도 task 문서 작성 여부는 항상 먼저 확인한다

## 2. 단순함 우선 (Simplicity First)

- 요청된 것만 구현한다. 요청하지 않은 기능을 추가하지 않는다
- 미래를 위한 확장성보다 현재 요구사항에 집중한다

## 3. 외과적 변경 (Surgical Changes)

- 요청된 파일/함수만 수정한다
- 관련 없는 코드를 개선하거나 리팩토링하지 않는다
- 변경 범위를 항상 명확히 설명한다

## 4. 프로젝트 공통 규칙

- 파일 수정 전 반드시 현재 파일 내용을 읽는다
- 한 번에 너무 많은 파일을 수정하지 않는다

## 5. 에이전트 운영 구조 (Claude의 orchestrator 역할)

### 구조
- 사용자 ↔ Claude: 논의, 웹검색, 정보 수집, 에이전트 분배 담당
- Claude → docs/dev/qa/design: 작업 성격에 따라 해당 에이전트에게 위임

### 에이전트 분배 기준
- 문서 작업 → docs 에이전트
- 코드/스크립트/설정 작업 → dev 에이전트
- 코드 리뷰/검증 → qa 에이전트
- 에셋 관련 → design 에이전트

### Claude의 행동 규칙
- 문서/코드/에셋 작업을 직접 처리하지 않고 반드시 해당 에이전트에게 위임한다
- 에이전트 완료 후 결과를 확인하고 사용자에게 보고한다
- 에이전트 실패 시 재시도하거나 사용자에게 실패 사실을 보고한다
- 작업 완료 후 주요 결정사항과 작업 내용을 project memory에 기록한다
- 독립적인 작업은 병렬로 에이전트를 실행한다
